হ্যাঁ, এবার আমরা **সাজানো enterprise pattern** দিয়ে শুরু করবো।
আমাদের goal হবে:

```text
কম code → বেশি module
একই table/page pattern → সব জায়গায় reuse
পরে বারবার code change/refactor না লাগে
RBAC আলাদা, Audit workflow আলাদা
```

আমি এখন থেকে AMS-এর Audit side এভাবে চালাবো।

---

# AMS Audit Development Master Plan

## Golden Rule

আমরা কোনো module আলাদা আলাদা random ভাবে বানাবো না।

প্রতিটা module এই same pattern follow করবে:

```text
1. Database model
2. Schema
3. Repository
4. Service
5. API Router
6. Permission seed
7. Navigation/Menu seed
8. Frontend API client
9. Reusable table
10. Right drawer form
11. Permission-based buttons
```

এতে Company, Branch, Audit Entity, Audit Assignment, Team, Finding—সব module same structure পাবে।

---

# First Big Decision

আমাদের AMS-এ **৩টা layer** থাকবে।

```text
1. Security Layer
   User, Role, Permission, Menu, Button

2. Master Data Layer
   Company, Branch, Department, Employee, Audit Entity, Project, Incident

3. Audit Workflow Layer
   Audit Assignment, Team, Visit, Checklist, Working Paper, Finding, Report
```

Security layer already good stage-এ আছে। এখন Master + Audit Workflow বানাবো।

এইগুলো একবার বানালে পরে সব module দ্রুত হবে।

# Code Reuse Strategy

## Backend

আমরা generic pattern রাখবো:

```text
BaseRepository
BaseService
PaginationParams
PaginatedResponse
Status enum
Audit fields mixin
```

তাহলে প্রতিটা module-এ শুধু module-specific logic লিখবো।

Example:

```text
CompanyRepository
AuditEntityRepository
AuditAssignmentRepository
AuditFindingRepository
```

সব repository common pagination/search/status filter use করবে।

---

## Frontend

সব page একই template follow করবে।

```tsx
<EnterprisePageHeader />
<EnterpriseFilterBar />
<EnterpriseDataTable />
<RightDrawer />
<ConfirmDialog />
```

Module শুধু data/config দিবে।

Example:

```tsx
const actions = useModuleActions("audit_assignment");
```

Then:

```tsx
actions.canCreate
actions.canUpdate
actions.canDelete
```

এটা Company, Branch, Audit Entity, Audit Assignment—সব জায়গায় same হবে।

# Final Decision

আমরা এখন থেকে এই rule follow করবো:

```text
আগে reusable foundation
তারপর master module
তারপর audit workflow
তারপর report/dashboard
```

এতে AMS clean থাকবে, কম code হবে, same pattern সব জায়গায় use হবে, আর future module add করতে গেলে শুধু model/schema/config বদলালেই হবে।
